# 웹 프로그래밍 과제 결과 보고서

## 1. 출판된 github.com 홈페이지 주소
*   **배포 주소**: https://kimqyoung.github.io/CWU_Web

## 2. 웹사이트 목적
*   단순히 텍스트를 나열하는 기존의 정적인 이력서를 탈피하고, **아이젠하워 매트릭스(중요도/긴급도)**와 **시간축(타임라인)**을 결합하여 나의 대학 생활과 프로젝트 포트폴리오를 시각적, 입체적으로 보여주는 **인터랙티브 개인 포트폴리오 웹사이트 제작**

## 3. 웹사이트 목표 및 기획 대비 변경점
*   **웹사이트 목표**:
    *   **A. 동적인 포트폴리오(로드맵) 제공**: 2차원 공간(사분면) 화면에서 마우스 드래그 앤 드롭(Drag & Drop) 및 크기 조정 기능으로 일정을 직접 배치하며 즐길 수 있는 동적 이력서 제공.
    *   **B. 대학생활 및 성과 시각화**: 전공 프로젝트, 알고리즘 스터디, 자격증(리눅스 마스터 등) 활동 결과를 Chart.js를 이용한 통계 차트와 연대기(Timeline) 형태로 시각화.
    *   **C. AI 프롬프트 추출**: 활동 내역을 AI 자기소개서 작성용 프롬프트 텍스트로 자동 변환해주는 편의 기능 제공.
*   **기획 대비 변경점 및 이유**:
    *   초기 기획에서는 별도의 백엔드 서버 구축 및 데이터베이스 연동을 고려했으나, 아직 백엔드 역량이 부족하여 웹 브라우저 내장 데이터베이스인 **'Local Storage'를 활용하는 자바스크립트 기반 방식으로 변경**했습니다.
    *   이를 통해 서버 유지 비용 없이 영구적으로 구동 가능하며, 평가 시 텅 빈 화면이 보이지 않도록 5개의 샘플 프로젝트 데이터와 프로필 정보를 스크립트 단에 하드코딩하여 기본값으로 나타나도록 완성도를 높였습니다.

## 4. 웹사이트 제작 주요 과정

### A. 웹사이트 목표에 맞게 HTML과 CSS, JS를 이용하여 웹사이트를 만들어 가는 과정
*   **데이터베이스의 부재 극복 (로컬 스토리지 채택)**: 백엔드 지식이 부족한 상태에서 브라우저를 껐다 켜도 사용자가 입력한 포트폴리오 데이터가 날아가지 않게 유지하는 방법을 깊이 고민했습니다. 수많은 인공지능 도구 서치와 시행착오 끝에, 브라우저 내장 API인 `Local Storage`를 미니 데이터베이스처럼 활용하는 아이디어를 알게 되었고, JSON 문자열 변환 기법(`JSON.stringify`, `JSON.parse`)을 적용해 입력 데이터를 영구 보존하는 데 성공했습니다.
*   **클릭과 드래그의 충돌 문제 해결**: 타임라인 위의 포트폴리오 항목을 '클릭'하여 팝업창을 띄우려고 했는데, 마우스가 1픽셀이라도 움직이면 '드래그' 이벤트로 우선 인식되어 원치 않게 일정이 이리저리 이동해버리는 심각한 조작 버그가 있었습니다. 이를 해결하기 위해 마우스를 누른 위치(`mousedown`)와 뗀 위치(`mouseup`)의 픽셀 좌표 차이를 자바스크립트로 계산하여, **마우스가 일정 픽셀 이상 움직였을 때만 드래그로 판정**하고 미세한 움직임은 단순 클릭으로 간주해 제자리로 되돌리도록 세밀하게 로직을 보완했습니다.
*   **PDF 내보내기 대신 AI 프롬프트 추출기 개발**: 당초 이력서와 활동 데이터를 모아 한 장의 깔끔한 PDF로 내보내는 기능을 기획했습니다. 그러나 외부 라이브러리 없이 JS만으로 복잡한 HTML 레이아웃과 CSS 스타일을 캡처하여 PDF로 렌더링하는 것은 제 현재 수준에서 기술적 난이도가 너무 높았습니다. 그래서 기획을 틀어서 입력된 모든 활동 데이터를 정형화된 자기소개서용 텍스트로 자동 결합해 주는 **'AI 프롬프트 추출 기능'**을 고안하였고, 버튼 클릭 한 번으로 사용자가 ChatGPT 등에게 이력서 생성을 지시할 수 있도록 대체했습니다.
*   **화면 레이아웃 및 렌더링 로직 최적화**: 단순히 HTML `<table>` 요소로 일정을 배치하려던 초기 계획을 폐기하고, CSS Grid를 베이스로 개별 속성에 `position: absolute`를 적용했습니다. 사용자의 화면 해상도나 확대 배율에 따라 픽셀(Pixel) 값이 달라져도 마우스 좌표값을 실제 날짜(YYYY-MM-DD)로 정확히 양방향 맵핑하는 수학적 알고리즘을 구현해 냈습니다.

### B. Github Page를 이용한 온라인 출판 과정
*   로컬 환경에서 테스트와 개발을 마친 HTML, CSS, JavaScript 소스코드 파일들을 Github를 이용하여 Github Repository(`CWU_Web`)에 Commit 및 Push 하였습니다.
*   이후 해당 레포지토리의 [Settings] - [Pages] 탭으로 이동하여 배포 브랜치(`main`)를 설정한 뒤 **Github Pages 기능을 활성화**하여, 누구든 인터넷만 연결되어 있다면 접속 가능한 공개 URL을 발급받아 성공적으로 배포를 완료했습니다.

### C. (최종 결과물로서의) 웹사이트 구현 내용
*   **인트로 및 메인 대시보드 (`index.html`)**: 프로젝트를 중요도(상/하)와 긴급도(좌/우)로 나누어 볼 수 있는 2차원 아이젠하워 사분면을 구현했습니다. 마우스 드래그를 이용해 빈 타임라인 공간을 스크롤 탐색하고, 박스 양끝을 잡아 기간을 조절하는 크기 조정 기능을 제공합니다. 또한 시스템 OS와 연동되는 다크 모드/라이트 모드 테마 변경 설정 팝업을 상단에 배치했습니다.
*   **포트폴리오 프로필 (`profile.html`)**: Chart.js 라이브러리를 활용하여 내가 등록한 목표들의 중요도 분포를 보여주는 도넛형 차트와, 진행 상황을 보여주는 차트를 구현해 통계 대시보드를 완성했습니다. 페이지 중앙에는 시간순 정렬된 **가로형 타임라인** UI를 제공하며, 우측에는 사용자의 학력과 기술 스택 등을 한눈에 보여주는 요약 카드를 배치했습니다.
*   **상세 소개 페이지 (`about.html`)**: 애플 웹사이트 스타일이 짙게 적용된 레이아웃을 바탕으로, 본 프로젝트의 3가지 핵심 기능(드래그 앤 드롭, 통계 차트, AI 프롬프트)을 이모지와 함께 요약하고, 개발자의 GitHub 링크를 돋보이게 배치한 페이지를 구현했습니다.

## 5. AI 도구를 활용했던 과정과 내용
*   **클린 코드 작성을 위한 작명 보조**: 개발 과정에서 개인적인 영어 어휘력 부족으로 인한 직관적이지 않은 변수나 함수 이름 때문에 코드의 목적을 한눈에 파악하기 힘든 문제가 있었습니다. 이때 어떤 역할을 하는 로직인지 AI에게 설명한 뒤, `loadRoadmapItems`, `setupCustomDropdown`, `viewportCenterDate` 등과 같이 목적에 맞는 영문 표기법으로 명확한 변수명과 함수명을 추천받아 가독성이 좋은 코드를 작성하는 데 큰 도움을 받았습니다.
*   **알고리즘 및 버그 디버깅**: 자바스크립트로 직접 DOM을 조작하고 마우스 좌표를 날짜로 매핑하는 계산 과정에서 아이템이 화면 밖으로 날아가는 버그나 드래그 이벤트 충돌 버그가 빈번하게 발생했습니다. 이때 AI 도구를 디버깅 보조 도구로 적극 활용하여 제가 짠 오류 코드를 분석시키고 최적의 알고리즘을 조언받아 적용할 수 있었습니다.
*   **UI/UX 조언 및 코드 리팩토링**: 기존의 딱딱한 UI에서 벗어나기 위해 AI와 대화하며 애플 웹사이트 스타일과 애니메이션 속성을 CSS 변수로 설계했습니다. 또한, 수백 줄에 달하는 가독성이 좋지 않은 코드를 분석하게 한 뒤 공통 기능(로컬 스토리지 입출력, 드롭다운 메뉴, 테마 변경 등)을 함수로 깔끔하게 묶어내는 리팩토링 과정을 거쳐 웹사이트의 전반적인 완성도와 코드 품질을 대폭 끌어올렸습니다.
